算法的时间复杂度与空间复杂度

发表于2023-09-05|更新于2023-09-06|编程
|阅读量:

算法是指用来操作数据、解决程序问题的一组方法。对于同一个问题，使用不同的算法，也许最终的结果是一样的，但是在过程中消耗的资源和时间却有很大的差别。衡量不同算法直接的优劣主要通过时间和空间两个维度去考量。

时间维度：是指执行当前算法所消耗的时间，我们通常用【时间复杂度】来描述

空间维度：是指执行当前算法需要占用多少内存空间，我们通常用【空间复杂度】来描述。

评价一个算法的效率主要看它的时间复杂度和空间复杂度，然而，有的时候时间和空间却又不可兼得，所以需要找一个平衡点。

所以，算法的效率=运行的时间+所需的存储空间

时间复杂度

先看个两个例子：

func1 函数有两行代码，共执行了 2 次

调用 func 函数，for 循环外有2行代码，循环体有 3 行代码，如果 n 为 1，共执行了 5 次；如果 n 为 2，共执行了 8次；如果 n 为 3，共执行了 11次；如果为 n，共执行 3n+2。

在一个算法的语句执行的次数称为语句频度或者时间频度，表示为 T(n)，n 表示问题的规模或者数量，T 代表输入 n 的时的总的次数。回到上面的func 函数代码，如果 n 为 1，表达为：T(1)=5，如果n为 2，T(2)=8，如果n为3，T(3)=11，如果为 n，T(n)=3n+2。func 函数执行次数就是：T(n)=2

有些代码不像例子中代码这么简单，可以一个个数。比如递归算法、两层嵌套for 循环、一个函数调用另外一个函数，这种使用 T(n)就比较麻烦，并且运算起来也麻烦，所以算法一般使用T(n) 简化估算值，来衡量代码执行的速度，这种算法的估算值叫做时间复杂度。

func1 函数的时间复杂度，func1 函数的总执行次数为 T(n)=2，如果 T(n)=常数，则可以将常数估算为 1，所以 func1 函数的时间复杂度是 1。

func 函数的总的执行数为 T(n)=3n+2，如果 T(n)= 常数n+ 常数，将常数估算为1，得到 T(n)= 1n+1，因为随着 n 越来越大，加号后面的常数不变，不会起决定性作用，没有意义，所以去除。所以，func 函数的时间复杂度为 T(n)=n。

举例，T(n)=5n3+6666n2+233，将常数估算为 1，得到T(n)=1n3+1n2+1，留下可能影响的最大的值，所以得到时间复杂度为 T(n)=n3

上面表达时间复杂度还不完整，在大 O 符号表示法中，时间复杂度的公式是：T(n)=O(f(n))，f(n) 表示函数执行的估算次数之和，所以上面例子的时间复杂度分别为：

func1 函数：T(n)=2=O(1)

func 函数：T(n)=3n+2=O(n)

T(n)=5n3+6666n2+233=O(n3)

总结时间复杂度的表示方法，T(n)是不是常数：

  1. 是：时间复杂度为 O(1)
  2. 否：时间复杂度为 O(保留 T(n)的最高次项并且去掉最高次项的系数)
